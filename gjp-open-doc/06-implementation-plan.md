# Implementation Plan

This file is the implementation plan map, not the full delivery plan.

Implementation planning is split into focused files under `implementation-plan/` so future phases, release tracks, and migration tasks can be added without making one long plan.

## Implementation Plan Structure

| File | Purpose |
| --- | --- |
| [implementation-plan/00-decisions.md](implementation-plan/00-decisions.md) | Decisions to confirm before implementation starts. |
| [implementation-plan/01-foundation.md](implementation-plan/01-foundation.md) | App foundation phase. |
| [implementation-plan/02-content.md](implementation-plan/02-content.md) | Core and media content phases. |
| [implementation-plan/03-polish-release.md](implementation-plan/03-polish-release.md) | Polish and release readiness phases. |
| [implementation-plan/04-milestones.md](implementation-plan/04-milestones.md) | Suggested milestones and first engineering task. |
| [implementation-plan/05-native-projects.md](implementation-plan/05-native-projects.md) | Native Android/iOS project creation plan. |
| [implementation-plan/_template.md](implementation-plan/_template.md) | Template for future implementation plan files. |

## How To Add Plan Items

1. Add or update a focused file under `implementation-plan/`.
2. Use one file per phase, release train, risk area, or milestone group.
3. Link new files in the Implementation Plan Structure table above.
4. Keep this root file as the plan index.
