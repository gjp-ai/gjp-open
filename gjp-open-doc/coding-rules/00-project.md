# Project Coding Rules

- Use TypeScript for app code.
- Keep `strict` TypeScript enabled.
- Do not use `any` unless a short comment explains why the external type cannot be known.
- Keep visible strings in i18n files.
- Use `EN` and `ZH` as the only language codes unless the API adds more.
- Use public API endpoints only.
- Keep secrets, signing files, keystores, provisioning profiles, and environment secrets out of source control.
