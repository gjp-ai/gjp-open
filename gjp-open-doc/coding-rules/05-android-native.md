# Native Android Coding Rules

- Use Kotlin only for app code unless a Java file is generated by tooling.
- Use Jetpack Compose for UI.
- Keep Composables stateless where practical; screen state belongs in ViewModels.
- Keep API DTOs separate from UI models when UI needs derived display fields.
- Use sealed state types for loading, content, empty, and error states.
- Use string resources for visible EN/ZH text.
- Use Material 3 theme tokens instead of hardcoded colors.
- Use Coil for network images.
- Use Media3 for video and audio.
- Add unit tests for API clients, repositories, and ViewModels.
- Add Compose UI tests for primary navigation and important screen states.
