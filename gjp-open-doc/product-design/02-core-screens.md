# Core Screens

## Websites

- Search input.
- Tag chips from app settings.
- Website cards with logo, name, description, tags, and visit action.
- External URLs open in the system browser.

## Q&A

- Search input.
- Tag chips from app settings.
- Collapsible or card-based question/answer layout.
- Copy/share answer can be considered after MVP.

## Articles

- Article cards with cover image, title, summary, source, tags, and updated date.
- Article detail screen with readable content width, source link, and cover image.
- HTML content must be sanitized or rendered by a trusted native HTML renderer.

## Images

- Grid layout.
- Thumbnail first, full image on preview.
- Full-screen preview supports close, share/open URL, and image metadata.

## Videos

- List cards with cover image and title.
- Dedicated video player using the API stream URL.
- Player must handle loading and playback errors.

## Audio

- Audio list cards with cover image, title, artist, subtitle, and tags.
- Dedicated audio player screen.
- Background playback is optional after MVP; do not promise it in the first release.

## Files

- File list cards with name, tags, and updated date.
- File action opens the API file URL through the OS.

## More / Settings

- Language toggle: EN / ZH.
- Theme toggle: light / dark / system if implemented.
- Theme color selection if retained from the web experience.
- App version, privacy policy, support email, and website link.
