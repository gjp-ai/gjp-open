# Product Design

This file is the product design map, not the full design catalog.

Design details are split into focused files under `product-design/` so future screens, journeys, and interaction specs can be added without creating one large document.

## Product Design Structure

| File | Purpose |
| --- | --- |
| [product-design/00-principles.md](product-design/00-principles.md) | Experience principles and visual direction. |
| [product-design/01-information-architecture.md](product-design/01-information-architecture.md) | App information architecture and navigation model. |
| [product-design/02-core-screens.md](product-design/02-core-screens.md) | Screen-level product behavior. |
| [product-design/03-states-accessibility.md](product-design/03-states-accessibility.md) | UI states, accessibility, and empty/error copy. |
| [product-design/_template.md](product-design/_template.md) | Template for future product design docs. |

## How To Add Product Design Docs

1. Add a focused file under `product-design/`.
2. Use one file per screen, journey, interaction pattern, or design system topic.
3. Link the new file in the Product Design Structure table above.
4. Split large screen specs by state or user journey when they grow beyond roughly 200-300 lines.
